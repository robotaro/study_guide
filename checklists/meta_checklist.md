# Meta ML Engineer — Interview Prep Checklist

Organized by interview round. See also: [Master Study Plan](study_plan.md)

---

## Round 1-2: Coding (2 rounds)

Focus: LeetCode-style problems, 45 min each. Typically 1 medium + 1 medium/hard per round.

### Patterns to drill
- [ ] Arrays & strings: two pointers, sliding window, prefix sums ([notes](../topics/coding/concepts/arrays.md))
- [ ] Binary trees: DFS, BFS, recursive construction ([notes](../topics/coding/concepts/binary_trees.md))
- [ ] Graphs: BFS, DFS, topological sort, connected components (no notes yet)
- [ ] Dynamic programming: 1D/2D DP, knapsack, subsequences ([notes](../topics/coding/concepts/dynamic_programming.md))
- [ ] Heaps: top-k, merge k sorted, median stream ([notes](../topics/coding/concepts/heaps.md))
- [ ] Binary search: on sorted arrays and on answer space (no notes yet)
- [ ] Hash maps: frequency counting, two-sum patterns (no notes yet)

### Practice targets
- [ ] Solve 20 medium problems across the above patterns ([leetcode tracker](../topics/coding/leetcode/README.md))
- [ ] Solve 5 hard problems for stretch practice ([leetcode tracker](../topics/coding/leetcode/README.md))
- [ ] Do 3 timed mock sessions (2 problems in 45 min)
- [ ] Review Meta-specific coding tips ([notes](../topics/coding/meta_tips.md))
- [ ] Review Python tips for clean, fast implementations ([notes](../topics/coding/python_tips.md))

---

## Round 3: ML System Design

Focus: design an end-to-end ML system for a product problem. ~45 min.

### Common Meta design questions to practice
- [ ] Design a news feed ranking system (no notes yet)
- [ ] Design a content moderation pipeline (text/image) (no notes yet)
- [ ] Design a recommendation system (ads, friends, groups) (no notes yet)
- [ ] Design a search ranking system (no notes yet)
- [ ] Design an integrity/harmful content detection system (no notes yet)

### Key concepts to review
- [ ] End-to-end ML pipeline: problem framing → features → model → serving → metrics
- [ ] Feature engineering at scale: embeddings, real-time features, feature stores
- [ ] Model selection tradeoffs: latency, accuracy, maintainability
- [ ] Online vs offline metrics: engagement rate, precision@k, NDCG
- [ ] A/B testing and experimentation framework
- [ ] Training pipeline: distributed training, data pipelines, retraining cadence

---

## Round 4: Behavioral

Focus: Meta values — Move Fast, Be Bold, Focus on Impact, Build Social Value, Be Open.

### STAR stories to prepare
- [ ] Story about driving a project with significant impact ([notes](../topics/behavioral/meta_behavioral.md))
- [ ] Story about resolving a conflict or disagreement with a teammate
- [ ] Story about a time you failed and what you learned
- [ ] Story about working under tight deadlines / moving fast
- [ ] Story about making a bold technical decision

### Standard questions
- [ ] "Why Meta?"
- [ ] "Tell me about yourself" (2-min version)
- [ ] "What's the most impactful project you've worked on?"
- [ ] "Tell me about a time you had to influence without authority"

### Prep actions
- [ ] Watch 3 Dan Croitor Meta behavioral videos ([notes](../topics/behavioral/meta_behavioral.md))
- [ ] Practice each story out loud, time to under 2 min
- [ ] Do 1 mock behavioral interview with a friend or recording
